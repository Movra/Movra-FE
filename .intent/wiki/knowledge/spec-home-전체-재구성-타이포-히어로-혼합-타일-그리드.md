# Spec: HOME 전체 재구성 — 타이포 히어로 + 혼합 타일 그리드

> status: 🚧 draft (awaiting human approval — review then `intent spec approve <slug>`)

## 문제 정의

[[spec-home-듀오링고풍-리디자인]](INT-001)로 빽빽한 7패널 격자를 없앴고, [[spec-home-히어로-깊이-톤-강화]](INT-002)로 깊이·톤을 넣었다. 그러나 INT-001 방식(히어로가 첫 화면을 통째로 채움 + 나머지를 1열 풀폭 카드로 쌓음)이 새로운 단조로움을 만들었다:
- 히어로 = 화면 하나를 통째로 먹는 거대 박스 1개.
- 아래로 시간표·주간목표·시험·Focus·요약이 비슷한 폭의 큰 박스 5개로 세로로 줄줄이.
- 즉 '박스가 너무 크면서 너무 많다' = 크기·리듬 변주가 없어 다시 AI(템플릿)스럽다.
- 추가로, 현재 일러스트 캐릭터 자체가 AI스럽게 보여 히어로의 시각 중심으로 부적합.

## 목표

frontend-design 관점으로 HOME 전체를 의도적 위계로 재구성한다: 히어로를 적당히 축소하고 캐릭터 대신 타이포 중심(헤드라인 + TopPick + 지금 시작)을 주인공으로, 그 아래를 크기가 차등화된 혼합 타일 그리드(요약·시간표·시험·목표)로 배치한다. INT-002의 깊이·톤은 유지한다. 범위: HOME 전체(src/features/core-loop/**).

## 가정 (확정)

- '적당히 나눠서' = (a) 원래의 균일 빽빽 격자 복귀도, (b) 현재의 거대 박스 타워도 아닌, 크기·중요도가 다른 박스를 의도적 위계로 배치하는 것.
- 히어로는 적당히 축소: 첫 화면을 통째로 채우지 않고, 아래 보조 타일 일부가 첫 화면에 살짝 보이게.
- 히어로의 새 시각 중심은 타이포(큰 헤드라인 + TopPick + 지금 시작). 큰 일러스트 캐릭터는 히어로에서 제거.
- 캐릭터는 히어로에서만 제거. 로딩·에러·회복·시간표 빈상태·Focus·요약카드 등 기능적 자리의 캐릭터는 유지.
- 박스 개수는 유지하되 크기·배치만 정리(주/보조 차등 + 작은 타일 묶음). 섹션을 새로 추가하지 않는다.
- INT-002 깊이·톤(canvas 레이어·헤어라인·소프트 섀도우·flow 칩) 유지. green-first 유지. 새 hue 금지.
- API 호출·응답 타입·쿼리 키 불변. 게임화(스트릭·포인트·랭킹·손실 프레이밍) 금지.

## 명시적 비목표

- ❌ 원래의 균일·빽빽 격자로 회귀 (모든 타일 같은 크기·중요도).
- ❌ 히어로를 다시 거대 박스로 두기.
- ❌ 히어로에 큰 일러스트 캐릭터 재등장.
- ❌ 새 섹션/위젯 추가로 정보량·복잡도 증가.
- ❌ 새 포인트 색, API/데이터 변경, HOME 외 페이지 변경.

## 경계 예시 (되는 것 / 안 되는 것)

- ✅ 히어로 높이를 화면의 60~70% 정도로 줄여 아래 타일이 살짝 보임 → 안 됨: 100dvh 꽉 채움 유지.
- ✅ 요약 지표를 작은 타일들로, 시간표를 더 큰 타일로 차등 배치(혼합 그리드) → 안 됨: 전부 동일 폭·높이 카드.
- ✅ 히어로에서 캐릭터 자리를 비우고 타이포·TopPick·진행으로 채움 → 안 됨: 같은 자리에 다른 일러스트 박아넣기.
- ✅ 빈상태/회복/요약의 작은 캐릭터는 그대로 → 안 됨: 그것들까지 전부 제거(이번 범위 밖).
- ✅ INT-002 톤·헤어라인 위에 그리드만 재배치 → 안 됨: 흰 배경·무톤으로 되돌리기.

## 성공 기준 (눈으로 확인)

1. 첫 화면에 축소된 히어로 + 아래 타일 일부가 함께 보인다(히어로가 100dvh를 꽉 채우지 않음). → 데스크톱/모바일 스크린샷.
2. 히어로에 큰 일러스트 캐릭터가 없고 타이포·TopPick·지금 시작이 시각 중심이다.
3. 보조 영역이 크기가 다른 혼합 타일(예: 시간표=넓게, 요약 지표=작은 타일들)로 위계가 읽힌다 — 균일 격자 아님.
4. 섹션/위젯 개수는 현재 대비 증가 없음(병합/제거 없이 재배치만 한 경우 동일).
5. INT-002 깊이·톤 유지, 새 hue 0건, 게임화·손실 어휘 0건, 무거운 그림자 없음.
6. 비-히어로 캐릭터(로딩/에러/회복/빈상태/요약)는 그대로 존재한다.
7. 타입체크·빌드·core-loop 테스트 통과. (web-design-guidelines로 접근성/UX 검수)
